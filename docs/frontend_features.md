# Docker Updater 前端功能总结文档

本规范文档详尽、全面地梳理了 Docker Updater (FNOS 容器升级管理器) 的前端所有视图、组件、底层交互机制及业务逻辑。该文档用于后续对后端 API 覆盖率、安全校验、竞态防护以及生命周期管理的审计。

---

## 1. 基础架构与体验层 (Infrastructure & Experience)

### 1.1 系统全局响应式布局
* **多端适配导航**：
  - **PC 端** (`md` 屏幕及以上)：左侧常驻 `135px` 宽度的极简白底侧边栏导航。
  - **移动端** (`md` 屏幕以下)：底部悬浮式磨砂毛玻璃底栏导航（高度 `16px` + 占位）。
* **布局高度与视口控制**：
  - 主内容区域在移动端强依赖动态视口高度类 `h-dvh`，防止移动端底部内容被浏览器/系统的底栏、地址栏遮挡。
  - 全局禁用通配符 `*` 惯性滚动，对于滚动区域精确应用 `touch-action` 规避系统级手势拦截冲突。

### 1.2 主题与本地化适配
* **自动深色模式**：
  - 前端监听系统级的深色模式媒体查询事件 (`(prefers-color-scheme: dark)`)。
  - 当检测到变化时，动态在页面 `<html>` 节点上挂载/卸载 `dark` 样式类，并重载 [Naive UI](https://www.naiveui.com/) 的 `darkTheme`。
  - 前端 UI 调色板由 CSS-first 配置驱动，实现无感平滑切换。
* **本地化与区域**：
  - 全局注入 Naive UI 中文包 (`zhCN` 与 `dateZhCN`)，日期解析默认为本地格式。

### 1.3 移动端手势与交互优化
* **防手势意外缩放**：
  - 拦截并阻止 `gesturestart` 默认行为。
  - 拦截 `touchstart` 中多于单指的触摸输入。
* **双击缩放过滤**：
  - 监听 `touchend` 动作，拦截 300 毫秒以内的双击缩放。但若点击的 Target 为输入元素（如 `INPUT`、`TEXTAREA`、`SELECT`、`isContentEditable`），则不予拦截，以确保移动端软键盘能随时调出。
* **空白失焦收键盘**：
  - 点击非输入框或按钮等非交互的空白区域时，自动触发 `document.activeElement.blur()`，收起软键盘并恢复界面常态。
* **路由切换焦点复位**：
  - 在 `router.afterEach()` 钩子中释放上一页面被点击元素的焦点，解决移动端按钮/底栏在切换页面后残留 Hover/Focus 高亮的问题。

## 2. 通信与长连接层 (Websocket Service)

前端基于 [wsService](/frontend/src/utils/websocket.ts) 维护单例 WebSocket 长连接，负责向页面及组件进行状态实时广播。

### 2.1 长连接生命周期与保活
* **连接端点**：优先选用 `wss:` 或 `ws:`，协议升级路径为 `/app/docker-updater/api/ws`。
* **心跳保活**：
  - 前端每 30 秒向后端发送一次 `{ type: 'ping' }` 帧。
  - 发送 ping 帧的同时启动 10 秒超时定时器 `pongTimeout`，若在 10 秒内未收到后端的 `{ type: 'pong' }` 回复，则主动触发 `ws.close()`，重新建立连接。
* **指数退避重连**：
  - 当触发 `onclose` 时，前端基于指数退避算法（配合随机抖动防止并发重打）执行自动重连，最大延迟设为 30 秒。
* **重新订阅恢复**：
  - 重连成功后，前端会遍历本地正在被监听的容器日志信道，自动向后端补发 `{"type": "subscribe", "target": "logs:container_name"}` 控制消息，防止升级过程中断连导致日志推送丢失。

### 2.2 发布-订阅通道
* **全局状态同步 (`type === 'status'`)**：
  - 后端直接广播最新的全局状态 payload（包含 containers 数组、last_check 检测时间、history 历史、当前 active 运行任务、queued 排队任务数组）。
  - **快照缓存机制**：前端在本地缓存最近一次的 `status` 数据。当用户在不同页面间来回切换并重新注册 `subscribeStatus` 时，前端立即用缓存快照触发 callback 完成初次渲染，消除全局 loading 骨架屏的等待感。
* **容器日志订阅 (`type === 'log'`)**：
  - 仅在前端下发订阅 `logs:{name}` 帧后被动接收。推送格式包含 `{container, task, message}`。前端多路分发给订阅了该容器日志的 Terminal 终端组件。
* **系统日志实时广播 (`type === 'syslog'`)**：
  - 全局广播，前端系统日志页 (`Logs.vue`) 订阅此通道，收到消息后实时在控制台尾部追加。

## 3. 页面功能规格 (View Specifications)

### 3.1 概览页面 (Dashboard)
* **状态指示卡片**：
  - 动态展示“待升级容器”数量、“已暂挂检测”容器数量。
  - 点击卡片可切换列表的过滤状态（`currentFilter: 'update' | 'deferred'`）。
* **上次检测时间**：
  - 显示宿主机最后一次轮询/手动检测镜像版本的时间，转换为本地可读时间。
* **手动触发检测**：
  - 点击“检测更新”按钮，调用 POST `/check` 接口，在后台异步触发宿主机容器镜像版本的全面轮询与哈希比对。
* **全选/半选批量处理**：
  - 支持列表中“待升级”容器的勾选。
  - 提供“全选”与“半选”状态，支持批量升级操作。
* **快捷列表卡片**（针对可更新容器）：
  - **一键升级**：触发所选容器更新流程（开启日志终端）。
  - **指定版本升级**：输入特定的 Tag 升级。
  - **暂挂检测**：设定在限定时间内不再对该容器镜像进行版本检测。
  - **取消暂挂**（仅限暂挂状态列表）：立即恢复检测。

### 3.2 容器列表页面 (Containers)
* **容器强制排序**：
  - 前端基于容器名称的字母顺序 (`localeCompare`) 进行强制升序排列，规避 WebSocket 状态广播频繁推送时因为顺序抖动造成页面卡片位置跳跃。
* **生命周期交互锁定 (Operating Lock)**：
  - 前端支持在卡片上直接对容器进行“启动”、“停止”、“重启”操作（调用对应 POST 接口 `/container/:name/[start|stop|restart]`）。
  - 为防止重复下发或者前后端操作竞态，前端在执行下发动作后，将操作目标记录到局部 Set 集合 `operatingContainers` 中，立即在 UI 上禁用该卡片对应的所有生命周期按钮，等待 WebSocket 广播回传最新的容器运行状态后自动解锁。
* **卡片操作集**：
  - **版本摘要比对区域**：卡片上动态比对“本地版本”与“最新版本”。优先展示可读性极佳的语义化版本号（如 `v1.2.0` $\rightarrow$ `v1.3.0`），在无语义版本时自动平滑回退展示短 Hash，鼠标悬浮（Hover）在按钮气泡上可随时查看完整 SHA Digest 摘要及进行一键复制。
  - **一键升级**：若检测到有新 Digest，点击可触发流式升级。
  - **指定版本升级**：弹窗输入具体 Tag（如特定版本号、镜像别名），允许不修改容器其他配置的情况下更换镜像版本标签进行升级。
  - **一键回滚**：当容器下挂有备份元数据时（即存在 `{name}_backup_docker_updater` 备份容器），显示回滚选项，弹出二次警告确认后，触发回滚。
  - **删除备份**：对有备份的容器，支持手动触发删除备份，调用 DELETE `/backup/:name` 释放存储空间，并清除前端的备份回滚按钮。
  - **运行日志诊断**：点击“日志”调用 GET `/container/:name/logs` 接口，拉取容器标准输出 (stdout) 与标准错误 (stderr) 的最近 200 行记录并进行模态框展示。
  - **暂挂与取消暂挂**。

### 3.3 镜像管理页面 (Images)
* **镜像统计**：
  - 统计宿主机上的“镜像总数”和“无标签残留镜像数”。
  - **虚悬镜像定义**：RepoTags 数组中包含 `<none>:<none>` 的镜像。
* **列表过滤**：
  - 可自由切换筛选“全部镜像”和仅“无标签残留镜像”。
* **一键安全 Prune**：
  - 点击“清理残留镜像”按钮，调用 POST `/images/prune` 接口，清理无标签且未被任何容器使用的虚悬镜像。
  - 成功后提示用户已释放的磁盘空间大小和清理镜像个数。
* **镜像强制删除**：
  - 支持卡片一键删除，调用 DELETE `/image`（传入 Query 参数 `id` 和 `force=true` 进行强制移除）。
* **镜像元数据展示**：
  - 展示镜像关联的 Container 容器名、架构类型 (`architecture`)、镜像 ID（支持截断并支持一键复制完整 SHA）、底栏独立语义版本徽章（若能够解析到版本号）、镜像总大小、创建日期。

### 3.4 升级历史页面 (History)
* **执行状态可视化**：
  - 卡片以绿色底打勾（升级成功）与红色底打叉（升级失败）区分展示升级结果。
  - 展示升级发生的精确时间以及升级后的完整镜像名称（提供一键复制按钮）。
* **升级物理日志回溯**：
  - 点击历史记录的“日志”按钮，调用 GET `/update-log/:name` 接口。
  - 若日志存在，则在 TerminalModal 终端中静态载入并展示该次升级流程的完整编译/部署日志。
* **单条历史记录物理删除**：
  - 点击“删除”，调用 DELETE `/history/:id`。物理删除指定 ID 的数据库记录及对应的 `/logs/${name}.log` 本地日志文件。
* **一键彻底清空历史**：
  - 点击“清空历史记录”，调用 DELETE `/history`。一键删除所有的数据库升级历史，并清空本地物理存储的 `/logs` 文件夹下所有日志。

### 3.5 任务队列页面 (Tasks)
* **当前活跃任务卡片 (Active Task)**：
  - 动态显示目前正在后台单工任务处理器中运行的任务（升级/回滚）。
  - 显示旋转 loading 态、任务名、入队时间、当前任务类型。
  - **日志流实时监控**：点击卡片的“日志”，首先调用静态日志 GET `/update-log/:name` 加载已输出内容，随后向 WebSocket 发送订阅请求，实时动态显示正在发生的拉流、编译及自愈日志。
* **排队任务列表 (Queued Tasks)**：
  - 以 `#1`, `#2` 序号展现当前积压在队列中的等待执行任务，显示对应的容器名与排队时间。
* **排队任务手动撤销**：
  - 对等待中的任务，支持点击“取消排队”，调用 POST `/tasks/cancel/:name` 从后台任务列表中移出。若任务已经转化为活动状态，则不允许移除并提示报错。

### 3.6 运行日志页面 (Logs)
* **快照加载与动态监听**：
  - 载入时首先调用 GET `/system/logs` 获取全局 info.log 日志文件的末尾 400 行。
  - 随后监听 WebSocket 的全局系统日志实时广播 (`syslog` 通道)，接收追加新写入的日志行。
* **日志前缀彩色高亮解析**：
  - 前端自主解析日志文本前缀（如 `[INFO]`, `[SUCCESS]`, `[WARN]`, `[WARNING]`, `[ERROR]`），提取时间戳、级别前缀和纯文本内容。
  - 自动渲染成对应的彩色 Badge 徽章和文本，改善文本阅读体验。
* **多级别细粒度过滤**：
  - 顶部常驻 Segment 过滤选项：ALL, INFO, SUCCESS, WARN, ERROR。切换后本地过滤符合级别条件的日志行。
* **关键字实时检索**：
  - 支持检索框输入，结合级别过滤条件，对日志行的内容进行本地模糊匹配检索。
* **一键清空系统日志**：
  - 点击“清空日志”按钮，调用 DELETE `/system/logs`。在后端以截断清空（保留空文件描述符）的形式清空物理 info.log 文件。
* **平滑滚动体验与悬浮回弹**：
  - 当接收到新的实时日志广播时，若滚动条处于底部附近，自动平滑滚动到底部展示新行。
  - 当用户向上滚动查看历史记录且偏离底部超过 `100px` 时，页面右下角淡入悬浮显示“回到底部”的动作按钮，点击后一键快速滚动至页面底端。

### 3.7 系统设置中心 (Settings)
* **参数配置表单**：
  - **备份功能控制**：开关 `backup_enabled`；开启后显示“备份自动清除周期”选项，可选择 `12`、`24`、`72`、`168`、`720` 小时。
  - **关联容器重启**：开关 `restart_stack`。
  - **更新轮询频率**：输入数值 `check_value` 配合单位 `check_type`（小时/天/周/月）。
  - **定时自动升级**：开关 `auto_update_enabled`。当开启时，定时扫描到更新将直接加入升级队列。
* **通知通道配置**：
  - **开关控制**：`notify_enabled` 决定是否开启推送。
  - **通知类型**：邮件通知 (`email`) 和 Webhook 推送 (`webhook`)。
  - **SMTP 邮件详情配置**：
    - 支持一键载入厂商预设配置（QQ邮箱、163网易、Gmail、Outlook）填充服务器与端口，或自定义配置。
    - 包含：SMTP服务器、端口、发件账号、授权码/密码、是否启用 SSL 加密开关、接收者邮箱。
    - 区分场景的自定义邮件模板：开启自动更新时设置升级成功/失败报告的主题与正文模板 (`smtp_subject_template` & `smtp_body_template`)；未开启自动更新时，设置更新检测发现新版本提醒的主题与正文模板 (`smtp_subject_template_check` & `smtp_body_template_check`)。
  - **Webhook 推送配置**：
    - 包含：Webhook URL、请求方法（POST/GET/PUT）。
    - 区分场景自定义 JSON Payload 模板。
    - 支持一键载入三大平台群机器人 Webhook Payload 预设：企业微信、钉钉、飞书、通用 JSON。
  - **模板变量说明**：前端提示用户，邮件和 Webhook 的模板文本/Payload 中均可使用 `{container_name}`、`{action_type}`、`{status}`、`{time}` 和 `{logs}` 进行变量动态插值替换。
  - **通知连通性测试**：
    - 点击发送测试按钮，调用 POST `/settings/test-email` 并携带当前表单配置（不依赖已存库的设定）。
    - 若邮件发送成功或 Webhook 响应成功，前端展示成功；若失败，捕获返回的后端原生 Error 以及 Webhook 目标平台的原始 Response 内容（做 `80` 字符截短展示），便于用户诊断配置。
* **仓库凭证管理 (Registry Credentials)**：
  - 列表展示已保存的凭证。出于安全机制，密码和 Token 在前端统一显示为 `******`。
  - 提供“添加凭证”和对已有凭证的“编辑”模态框。表单包含：仓库域名、用户名、密码/Token。
  - 支持一键物理删除指定凭据，调用 DELETE `/registries/:id`。
* **镜像加速源配置 (Mirrors)**：
  - 列表展示当前配置的镜像加速源。
  - 支持添加镜像加速源，并在弹窗中贴心提供国内常用加速镜像地址（如 DaoCloud 等）一键填入并添加。
  - 支持在列表中直接删除加速源项。
* **同步反馈与静默自保存机制 (Auto Save)**：
  - 除了弹窗类型的表单需要点击“保存”按钮外，页面内的所有常规 Switch 开关、Select 选择框、输入框（在失焦 `@blur` 时）发生数值改变，前端都会**自动触发**配置同步保存，向后端 POST `/settings`。
  - 配合自动保存，前端右上角会淡入显示带有 `正在同步...` 和 `配置已自动保存` 状态的过渡气泡反馈，提供微动交互体验，防止用户忘记点击保存或页面跳转导致配置丢失。

## 4. 前端交互核心模态框组件 (Core Components)

### 4.1 终端日志框 (TerminalModal)
* 模拟黑底绿字的 Linux 物理终端排版。
* 包含“自动锚定底部”特性，只要 `logRunning === true`（处于实时追加模式），容器内容发生变更时便会自动滚动到底部。
* 关闭弹框时，自动触发传入的 `unsubscribe` 取消订阅 WS 动作。

### 4.2 暂挂检测选择框 (DeferModal)
* 弹出选择框，提供 7天、14天、30天、90天等预设天数。
* 用户确认后，前端自动向后台计算或发送具体的暂挂周期天数参数。

### 4.3 诊断日志框 (DiagnosticsModal)
* 以无折行、允许自由复制的黑底灰字状态，静态展示从宿主机 Docker 引擎获取的容器 200 行历史输出。

### 4.4 指定版本升级框 (UpdateVersionModal)
* 展示当前容器的原始镜像名称。
* 提供输入框，允许用户自定义升级的 Tag。支持输入如 `latest`、`v1.2.3` 或其他特定镜像名，点击提交后调用携带 `target_image` 变量的更新流程。
